# Лабораторная работа: разработка Web3 dApp для Stacks

## 1. Цель работы
Разработать и подготовить к публикации децентрализованное Web3-приложение, интегрированное с ранее созданным fungible token контрактом в сети Stacks.

В рамках данной работы реализован **вариант 3: простой пул ликвидности** для пары **Lab Token / STX** с возможностью:

- подключать кошелёк пользователя;
- отображать баланс STX, баланс токена и баланс LP-токена;
- выполнять mint / transfer / burn для базового токена;
- добавлять ликвидность в пул;
- выводить ликвидность из пула;
- обменивать `STX -> token` и `token -> STX`;
- публиковать статический frontend через GitHub Pages.

- Сеть: `Stacks Testnet`
- ChainId: `2147483648` (`0x80000000`)
- RPC / API: `https://api.testnet.hiro.so`

## 4. Краткая архитектура решения
Решение состоит из двух смарт-контрактов и одного статического web-приложения:

1. `contracts/lab-token-2.clar`
   Базовый fungible token, используемый в приложении и в пуле ликвидности.

2. `contracts/liquidity-pool.clar`
   Упрощённый AMM-контракт для пары `Lab Token / STX`, который хранит резервы пула и выпускает LP-токен.

3. `index.html` + `src/main.ts` + `src/styles.css`
   Клиентское приложение, работающее через `@stacks/connect` и `@stacks/transactions`.

## 5. Что было изменено в репозитории
Ниже перечислены все основные изменения, которые были внесены в проект в ходе выполнения лабораторной работы.

### Шаг 1. Доработка базового токен-контракта
Файл: `contracts/lab-token-2.clar`

Что изменено:
- добавлены read-only функции:
  - `get-decimals`
  - `get-token-uri`
  - `get-total-supply`
  - `get-balance`
- доработана логика `transfer`:
  - теперь функция принимает `sender`, `recipient`, `memo`;
  - это позволяет пулу ликвидности вызывать transfer внутри своего контракта;
- `mint` и `burn` приведены к единообразному виду с ответом `(ok true)`;
- сохранена модель owner-only mint.

Зачем это понадобилось:
- фронтенду теперь есть что читать через read-only вызовы;
- пул ликвидности может безопасно работать с токеном без отдельного backend;
- контракт стал удобнее для интеграции в полноценный dApp.

### Шаг 2. Добавление контракта пула ликвидности
Файл: `contracts/liquidity-pool.clar`

Что реализовано:
- хранение резервов:
  - `token-reserve`
  - `stx-reserve`
- внутренний LP-токен:
  - `define-fungible-token lp-token`
- read-only функции:
  - `get-token-reserve`
  - `get-stx-reserve`
  - `get-lp-total-supply`
  - `get-liquidity-balance`
  - `get-pool-state`
  - `quote-add-liquidity`
  - `quote-remove-liquidity`
  - `quote-stx-for-token`
  - `quote-token-for-stx`
- основные public функции:
  - `add-liquidity`
  - `remove-liquidity`
  - `swap-stx-for-token`
  - `swap-token-for-stx`

Принятые упрощения:
- использована учебная модель **constant-product AMM**;
- комиссии пула не добавлялись;
- расчёты выполняются в микровеличинах без сложной математики и fee-механики;
- LP-токен хранится внутри контракта пула, что уменьшает количество зависимостей и упрощает демонстрацию.

## 6. Описание реализованных контрактов

### 6.1. Контракт `lab-token-2`
Основные функции:
- `mint(amount, recipient)`  
  Выпуск токенов. Доступен только owner.
- `transfer(amount, sender, recipient, memo)`  
  Перевод токенов между адресами. Используется и напрямую в UI, и из контракта пула.
- `burn(amount)`  
  Сжигание части токенов пользователем.
- `get-balance(account)`  
  Возвращает баланс указанного адреса.
- `get-total-supply()`  
  Возвращает общий supply токена.

### 6.2. Контракт `liquidity-pool`
Основные функции:
- `add-liquidity(token-amount, stx-amount)`  
  Добавляет активы в пул и выпускает LP-токены.
- `remove-liquidity(lp-amount)`  
  Сжигает LP-токены и возвращает пользователю долю токена и STX.
- `swap-stx-for-token(stx-in, min-token-out)`  
  Выполняет обмен STX на токен.
- `swap-token-for-stx(token-in, min-stx-out)`  
  Выполняет обмен токена на STX.

Read-only функции:
- `get-token-reserve()`
- `get-stx-reserve()`
- `get-lp-total-supply()`
- `get-liquidity-balance(account)`
- `quote-add-liquidity(token-amount)`
- `quote-remove-liquidity(lp-amount)`
- `quote-stx-for-token(stx-in)`
- `quote-token-for-stx(token-in)`

## 7. Возможности web-приложения
После запуска приложения пользователь может:

1. Подключить Stacks-кошелёк.
2. Указать адрес развернутых контрактов.
3. Просмотреть:
   - баланс STX;
   - баланс токена;
   - баланс LP;
   - текущие резервы пула.
4. Выполнить transfer токенов.
5. Выполнить burn токенов.
6. Если подключён owner-кошелёк, выполнить mint.
7. Добавить ликвидность в пул.
8. Вывести ликвидность из пула.
9. Выполнить обмен между STX и токеном в обоих направлениях.
